# 作业级分层一致性FUSE文件系统 全套测试验证方案

# 1 测试概述

## 1\.1 测试目标

本方案针对**作业级分层一致性FUSE文件系统**定制验证体系，核心验证四大核心特性正确性与稳定性，同时完成性能基线对标：

1. **分层一致性语义合规**：严格验证「作业内强线性一致、作业外弱一致、作业结束全局收敛」的核心分层模型符合设计规范。

2. **代理转发机制可靠**：验证单资源单宿主抢占、跨客户端IO代理转发、串行执行逻辑无误，从根源消除多客户端并发竞态。

3. **PRELOAD无侵入可控**：验证Coherent库按需加载、作业上下文识别、非作业路径透明透传，无业务侵入、无额外异常。

4. **容错与收敛完备**：验证节点故障、作业异常退出、锁超时、缓存失效等场景下，资源自动回收、数据无损、全局收敛正常。

5. **性能可控可接受**：作业内强一致仅引入极小转发开销，远优于全局强一致方案，作业外性能对齐原生FUSE弱一致基线。

## 1\.2 测试核心依据

测试依据项目架构设计、分层一致性定义、POSIX文件系统语义、HPC批量作业存储范式、主流FUSE一致性测试标准制定，覆盖学术界分层缓存、租约代理、作业域隔离、生命周期收敛等核心理论约束。

## 1\.3 测试范围

覆盖功能正确性、一致性语义、代理转发、锁机制、缓存管控、生命周期收敛、异常容错、压力稳定性、性能基线、兼容性全场景，区分**作业内强一致域**、**作业外弱一致域**、**作业收敛阶段**三类核心场景。

# 2 测试环境与前置准备

## 2\.1 集群环境拓扑

搭建多节点FUSE集群，模拟批量作业调度场景：

- 客户端节点：5台（Client1～Client5），统一挂载同一FUSE共享存储

- 作业集群：Job0绑定Client1～Client3（作业内强一致域）

- 外部节点：Client4、Client5（作业外弱一致域）

- 后端：统一分布式共享存储，保证全局数据最终落地唯一

- 调度环境：模拟SLURM/LSF作业调度，支持JobID注入、作业生命周期管控

## 2\.2 软件环境

- 底层：标准FUSE客户端，默认writeback缓存、原生弱一致策略

- 增强层：Coherent PRELOAD一致性库，支持动态加载/卸载、Job上下文识别

- 测试工具：fsstress、fsx\-linux、pjd\-fstest、fio、自定义并发校验脚本、md5全局校验工具

- 观测工具：RPC日志、锁状态监控、缓存超时监控、文件属性时序采集工具

## 2\.3 环境基线配置

- 开启FUSE默认缓存策略（entry\_timeout、attr\_timeout默认值）

- 支持动态切换PRELOAD开启/关闭，用于对照组基线对比

- 所有节点时间同步，保证操作时序可追溯
